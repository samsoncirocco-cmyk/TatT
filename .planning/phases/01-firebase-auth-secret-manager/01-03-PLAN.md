  <action>
    1. **Install dependency:**
       ```bash
       npm install @google-cloud/secret-manager
       ```

    2. **src/lib/secret-manager.ts** — Create Secret Manager wrapper:
       - Import `SecretManagerServiceClient` from `@google-cloud/secret-manager`
       - Create singleton client instance
       - `getSecret(secretName: string): Promise<string>` function:
         - Check in-memory cache first (Map<string, string>)
         - Build name: `projects/${process.env.GCP_PROJECT_ID}/secrets/${secretName}/versions/latest`
         - Call `client.accessSecretVersion({ name })`
         - Extract payload: `version.payload.data.toString('utf8')`
         - Cache and return
         - On error: fall back to `process.env[secretName]` if available (for local dev with .env.local)
         - If neither available: throw clear error: `Secret '${secretName}' not found in Secret Manager or environment variables`
       - `loadSecrets(names: string[]): Promise<Record<string, string>>` function:
         - Loads multiple secrets in parallel using Promise.all
         - Returns object mapping name to value
       - Export both functions
       - NOTE: This file is server-only (no 'use client', no NEXT_PUBLIC_ exposure)

    3. **server.js** — Remove hardcoded dev token:
       - Find line: `const FRONTEND_AUTH_TOKEN = process.env.FRONTEND_AUTH_TOKEN || 'dev-token-change-in-production';`
       - Replace with: `const FRONTEND_AUTH_TOKEN = process.env.FRONTEND_AUTH_TOKEN;`
       - Add startup check: if `!FRONTEND_AUTH_TOKEN`, log a WARNING but don't crash (allows local dev without token if using Firebase auth instead)
       - Find the console.log line comparing against 'dev-token-change-in-production' and update it to check if token is set: `${FRONTEND_AUTH_TOKEN ? 'Configured' : 'Not set (using Firebase auth)'}`
       - NOTE: server.js is the Express proxy that still runs alongside Next.js. It will eventually be deprecated in Phase 2 (Cloud Run migration). For now, just remove the hardcoded fallback.
  </action>

    - `@google-cloud/secret-manager` is in package.json dependencies
    - secret-manager.ts exports `getSecret` and `loadSecrets`
  </verify>
