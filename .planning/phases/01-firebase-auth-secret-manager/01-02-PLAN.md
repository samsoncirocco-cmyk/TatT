  <action>
    1. **Install dependency:**
       ```bash
       npm install next-firebase-auth-edge
       ```

    2. **middleware.ts** (project root, NOT in src/) — Create Edge-compatible auth middleware:
       - Import `authMiddleware` from `next-firebase-auth-edge/lib/next/middleware`
       - Configure with Firebase service account credentials from env vars:
         - `apiKey`: `process.env.NEXT_PUBLIC_FIREBASE_API_KEY`
         - `cookieName`: `'AuthToken'`
         - `cookieSignatureKeys`: `[process.env.AUTH_COOKIE_SIGNATURE_KEY_CURRENT!, process.env.AUTH_COOKIE_SIGNATURE_KEY_PREVIOUS!]`
         - `cookieSerializeOptions`: httpOnly, secure in production, sameSite 'lax', maxAge 12 days
         - `serviceAccount`: projectId, clientEmail, privateKey from env vars (replace \\n in privateKey)
       - `loginPath`: `/api/login`
       - `logoutPath`: `/api/logout`
       - `handleValidToken`: Return `NextResponse.next()` with headers (allow request)
       - `handleInvalidToken`: For API routes (`/api/v1/*`), return 401 JSON. For pages, redirect to `/login`
       - `handleError`: Log error, redirect to `/login` for pages, return 401 for API routes
       - `config.matcher`: `['/api/v1/:path*', '/dashboard/:path*', '/generate/:path*', '/smart-match/:path*', '/visualize/:path*', '/api/login', '/api/logout']`
       - Do NOT match: `/`, `/login`, `/signup`, `/api/health`, `/api/neo4j/:path*`, static assets

    3. **src/lib/auth-dal.ts** — Create Data Access Layer for auth:
       - Import `getTokens` from `next-firebase-auth-edge`
       - Import `cookies` from `next/headers`
       - `verifyAuth()` function: calls `getTokens(cookies(), { ...config })` with same Firebase config
       - Returns decoded token (uid, email, etc.) or throws Error('Unauthorized')
       - This is the CANONICAL auth check — every server-side data access must call this
       - Export type `AuthUser` with `uid`, `email`, `emailVerified` fields

    4. **src/lib/api-auth.ts** — Update to use Firebase token verification:
       - Keep function signature `verifyApiAuth(req: NextRequest)` for backward compatibility
       - REPLACE bearer token logic with Firebase token extraction from cookie
       - Import `getTokens` from `next-firebase-auth-edge`
       - Extract tokens from request cookies (middleware already handles setting cookies)
       - If tokens valid, return null (success) — same contract as before
       - If tokens invalid, return 401 NextResponse — same contract as before
       - REMOVE the `FRONTEND_AUTH_TOKEN` constant and `'dev-token-change-in-production'` fallback
       - Add optional `getUserFromRequest(req)` export that returns decoded user info (uid, email)
       - NOTE: During migration, middleware handles auth for /api/v1/* routes, so verifyApiAuth becomes a defense-in-depth check. Keep it but update the mechanism.
  </action>
  <verify>

    - Verify middleware.ts exports `middleware` and `config`
    - Verify auth-dal.ts exports `verifyAuth` and `AuthUser` type
    - Verify api-auth.ts no longer contains 'dev-token-change-in-production'
    - Verify `next-firebase-auth-edge` is in package.json dependencies
  </verify>
  <done>
    Edge middleware protects /api/v1/* and protected page routes. DAL provides verifyAuth() for server-side auth. api-auth.ts uses Firebase tokens instead of shared bearer token.
