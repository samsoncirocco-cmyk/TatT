<task type="auto">
  <name>Task 1: Create Pino logger with GCP severity mapping</name>
  <files>src/lib/logger.ts</files>
  <action>
Install pino: `npm install pino`

Create `src/lib/logger.ts` with:

1. Pino logger instance configured for GCP Cloud Logging:
   - `messageKey: 'message'` (GCP expects `message`, not `msg`)
   - `timestamp: pino.stdTimeFunctions.isoTime`
   - Custom `formatters.level()` that maps Pino levels to GCP severity strings:
     - trace/debug → 'DEBUG', info → 'INFO', warn → 'WARNING', error → 'ERROR', fatal → 'CRITICAL'
   - Custom `formatters.log()` that maps OpenTelemetry trace fields to GCP fields:
     - `trace_id` → `logging.googleapis.com/trace`
     - `span_id` → `logging.googleapis.com/spanId`
     - `trace_flags === '01'` → `logging.googleapis.com/trace_sampled`

2. Export a typed `logger` instance.

3. Export a helper function `createRequestLogger(routeName: string)` that returns an object with:
   - `start(event_type: string, metadata: Record<string, unknown>)` — logs at INFO, records start timestamp
   - `complete(event_type: string, metadata: Record<string, unknown>)` — logs at INFO with `duration_ms` computed from start
   - `error(event_type: string, err: Error, metadata?: Record<string, unknown>)` — logs at ERROR with full Error object (`{ err }` so Pino serializes stack trace)

This helper ensures consistent event shape across all routes without copy-pasting timing logic.

Do NOT use `@google-cloud/pino-logging-gcp-config` — the manual severity mapping is simpler and avoids an extra dependency. The research confirms the manual approach is sufficient.

Do NOT log at DEBUG level in production (Cloud Logging charges per GB ingested). Default log level should be 'info'.
  </action>

  </verify>

- Log `embeddings.started` with: `{ input_type, input_count }`
- Log `embeddings.completed` with: `{ embedding_count, dimension, duration_ms }`
