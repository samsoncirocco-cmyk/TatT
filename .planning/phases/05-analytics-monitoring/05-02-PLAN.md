**1. Create `scripts/setup-bigquery-sink.sh`:**

Bash script that sets up BigQuery dataset and Cloud Logging sink. Must be idempotent (safe to run multiple times).

```bash
#!/usr/bin/env bash
set -euo pipefail

PROJECT_ID="${GCP_PROJECT_ID:?GCP_PROJECT_ID must be set}"
DATASET="api_events"
SINK_NAME="api-events-sink"
LOCATION="us-central1"

# Create BigQuery dataset (idempotent)
bq mk --dataset \
  --location="$LOCATION" \
  --description="TatTester API event logs for analytics" \
  --default_table_expiration=7776000 \
  "$PROJECT_ID:$DATASET" 2>/dev/null || echo "Dataset $DATASET already exists"

# Create log sink routing structured events to BigQuery
# Filter matches all event_type fields from instrumented API routes
gcloud logging sinks create "$SINK_NAME" \
  "bigquery.googleapis.com/projects/$PROJECT_ID/datasets/$DATASET" \
  --project="$PROJECT_ID" \
  --log-filter='jsonPayload.event_type=~"^(generation|match|council|embeddings|budget)\."' \
  --description="Routes API events to BigQuery for analytics" \
  2>/dev/null || echo "Sink $SINK_NAME already exists, updating..." && \
  gcloud logging sinks update "$SINK_NAME" \
    "bigquery.googleapis.com/projects/$PROJECT_ID/datasets/$DATASET" \
    --project="$PROJECT_ID" \
    --log-filter='jsonPayload.event_type=~"^(generation|match|council|embeddings|budget)\."'

# Grant sink writer identity BigQuery Data Editor permission
WRITER_IDENTITY=$(gcloud logging sinks describe "$SINK_NAME" --project="$PROJECT_ID" --format='value(writerIdentity)')
gcloud projects add-iam-policy-binding "$PROJECT_ID" \
  --member="$WRITER_IDENTITY" \
  --role="roles/bigquery.dataEditor" \
  --condition=None \
  --quiet

echo "BigQuery sink setup complete."
echo "  Dataset: $PROJECT_ID:$DATASET"
echo "  Sink: $SINK_NAME"
echo "  Writer: $WRITER_IDENTITY"
```

Set `--default_table_expiration=7776000` (90 days in seconds) per research recommendation.

**2. Create `scripts/setup-monitoring.sh`:**

Bash script that creates log-based metrics, alert policies, and a dashboard. Must be idempotent.

Create 3 log-based counter metrics via `gcloud logging metrics create`:
- `api_errors` — filter: `severity="ERROR" AND jsonPayload.event_type=~".*\.failed"` — counts all API failures
- `generation_count` — filter: `jsonPayload.event_type="generation.completed"` — counts completed generations
- `budget_limit_reached` — filter: `jsonPayload.event_type="budget.limit_reached"` — counts budget limit hits

Create 1 log-based distribution metric via `gcloud logging metrics create`:
- `api_latency_ms` — filter: `jsonPayload.event_type=~".*\.completed" AND jsonPayload.duration_ms>0` — distribution metric on `jsonPayload.duration_ms` field with value extractor `EXTRACT(jsonPayload.duration_ms)` — this enables p50/p95/p99 percentile queries

Create a notification channel (email) using `gcloud beta monitoring channels create`:
- Type: email
- Address from `$ALERT_EMAIL` env var (required)

Create 3 alert policies using `gcloud alpha monitoring policies create` with inline JSON:
- **Budget 50%:** Threshold on `custom.googleapis.com/budget/spent_cents` > 25000 (=$250)
- **Budget 75%:** Threshold > 37500 (=$375)
- **Budget 90%:** Threshold > 45000 (=$450)
- All use `duration: 60s`, `comparison: COMPARISON_GT`, `perSeriesAligner: ALIGN_MAX`
- All notify via the email channel created above

Create dashboard JSON and apply via `gcloud monitoring dashboards create`:
- Use the dashboard JSON from the research (05-RESEARCH.md "Cloud Monitoring Dashboard JSON" section)
- Tiles:
  - API Error Rate (xyChart showing error count over time)
  - API Latency (xyChart showing p50/p95/p99 percentiles from `api_latency_ms` distribution metric)
  - Budget Spend scorecards with 50/75/90% thresholds
- Dashboard name: "TatTester API Metrics"
- For the API Latency xyChart tile, use `perSeriesAligner: ALIGN_DELTA` and `crossSeriesReducer: REDUCE_PERCENTILE_50/95/99` on the `api_latency_ms` distribution metric

Add a weekly BigQuery scheduled query SQL file at `scripts/sql/weekly-match-quality.sql`:
- The SQL from research section "Weekly Match Quality Report Query"
- Include a comment header explaining how to set up as scheduled query in BigQuery Console

Print instructions at end for manual steps (creating scheduled query in Console).

Make both scripts executable: `chmod +x scripts/setup-bigquery-sink.sh scripts/setup-monitoring.sh`
