**Create `src/services/storage/StorageFactory.ts`:**

```typescript
import { IDesignStorage } from './IDesignStorage';
import { LocalStorageAdapter } from './LocalStorageAdapter';
import { FirestoreAdapter } from './FirestoreAdapter';
import { getFirestore } from 'firebase/firestore';
```

- Singleton pattern: cache adapter instances to avoid recreating on every call
- `let cachedAdapter: { userId: string | null; adapter: IDesignStorage } | null = null;`

- `createStorageAdapter(user: { uid: string } | null): IDesignStorage`
  - If user is null or undefined → return `new LocalStorageAdapter()`
  - If user exists → return `new FirestoreAdapter(getFirestore())`
  - Cache by user.uid to avoid recreating adapter on every render
  - When user changes (login/logout), invalidate cache

- `getStorageAdapter(): IDesignStorage`
  - Returns cached adapter or LocalStorageAdapter as default
  - This is for use in non-React contexts (services) that can't access hooks

- `setCurrentUser(user: { uid: string } | null): void`
  - Updates the cached adapter when auth state changes
  - Call this from AuthContext/AuthProvider on auth state change

**Create `src/services/storage/firestoreZustandStorage.ts`:**

Custom Zustand `StateStorage` adapter that syncs store state to Firestore:

```typescript
import type { StateStorage } from 'zustand/middleware';
import { doc, getDoc, setDoc } from 'firebase/firestore';
import { getFirestore } from 'firebase/firestore';
```

- `createFirestoreZustandStorage(userId: string, designId: string): StateStorage`
  - `getItem(name)`: Read from `users/${userId}/designs/${designId}` doc field `storeState`, return JSON string or null
  - `setItem(name, value)`: Write to same doc with `setDoc(ref, { storeState: JSON.parse(value), updatedAt: new Date() }, { merge: true })`
  - `removeItem(name)`: Set `storeState` to null with merge
  - All methods async, wrapped in try-catch with console.error logging
  - Debounce setItem with 1000ms delay to avoid excessive Firestore writes during rapid layer edits (transforms, drags). Use a simple `setTimeout`/`clearTimeout` pattern — NOT lodash.

**Migrate `src/services/versionService.ts`:**

The existing service uses `safeLocalStorageGet`/`safeLocalStorageSet` directly. Refactor to use `IDesignStorage`:

- Add parameter `storage: IDesignStorage` to all exported functions (or accept it in a factory/init function)
- Better pattern: Create a `VersionService` class that takes `IDesignStorage` in constructor:

```typescript
export class VersionService {
  constructor(private storage: IDesignStorage, private userId: string) {}

  async getVersions(designId: string, limit?: number): Promise<DesignVersion[]> { ... }
  async addVersion(designId: string, data: Partial<DesignVersion>): Promise<DesignVersion> { ... }
  async branchFromVersion(designId: string, versionId: string): Promise<{ newDesignId: string, newVersionId: string }> { ... }
  async compareVersions(designId: string, v1Id: string, v2Id: string): Promise<VersionDiff> { ... }
  async deleteVersion(designId: string, versionId: string): Promise<void> { ... }
}
```

- `getVersions()` → calls `this.storage.loadVersions(this.userId, designId, limit)`
- `addVersion()` → creates version object (preserve existing `createVersion` logic), calls `this.storage.saveVersion()` and `this.storage.saveLayers()`. Enforces MAX_VERSIONS (50) by deleting oldest non-favorite if over limit.
- `branchFromVersion()` → loads source version and its layers, creates new design via `this.storage.saveDesign()`, saves cloned version with `branchedFrom` metadata, saves cloned layers with new IDs (use `generateLayerId()` from layerUtils). Uses optimistic approach (not Firestore transaction from client side — transactions are for server-side concurrent access).
- `compareVersions()` → loads both versions + layers, computes diff (preserve existing comparison logic: layer count diff, prompt diff, parameter diff, similarity score)
- `deleteVersion()` → calls `this.storage.deleteVersion()`
- Remove `purgeOldHistories()` (localStorage-specific cleanup) — Firestore doesn't need this
- Keep `generateVersionId()` as private method
- Keep `createVersion()` internal helper

**Also export standalone functions** as wrappers for backward compatibility (so existing callers don't all need updating at once):

```typescript
// Backward-compatible exports
export function getVersions(sessionId: string): DesignVersion[] { ... }
```

These wrappers use `getStorageAdapter()` from StorageFactory and the current userId. They're synchronous wrappers around async methods using a cached last-result pattern for the synchronous callers that exist in the codebase. Add a `// TODO: Migrate callers to async VersionService` comment.
