**LocalStorageAdapter** ā€” wraps existing localStorage patterns from versionService.ts:

- `implements IDesignStorage`
- Constructor: no args needed
- Storage keys: `design_${userId}_${designId}` for designs, `versions_${userId}_${designId}` for versions, `prefs_${userId}` for preferences
- Uses `JSON.parse`/`JSON.stringify` with try-catch (match existing `safeLocalStorageGet`/`safeLocalStorageSet` pattern from `src/services/storageService.ts`)
- `listDesigns()` scans localStorage keys with prefix `design_${userId}_`
- `saveLayers()` embeds layers in version object (localStorage has no subcollection concept)
- `loadLayers()` loads version and returns its layers array
- All methods are async (return Promises) for interface compatibility even though localStorage is synchronous
- `loadPreferences()` returns defaults if no saved preferences

**FirestoreAdapter** ā€” uses subcollection architecture per research:

- `implements IDesignStorage`
- Constructor takes `Firestore` instance: `constructor(private db: Firestore)`
- Import from `firebase/firestore`: `doc, setDoc, getDoc, getDocs, deleteDoc, collection, query, orderBy, limit, writeBatch`
- Firestore paths:
  - Designs: `users/${userId}/designs/${designId}`
  - Versions: `users/${userId}/designs/${designId}/versions/${versionId}`
  - Layers: `users/${userId}/designs/${designId}/versions/${versionId}/layers/${layerId}`
  - Preferences: `users/${userId}/preferences/settings`
- `saveDesign()` ā€” `setDoc` with merge:true for design metadata (createdAt, updatedAt, currentVersionId, bodyPart, canvas, isFavorite). Do NOT save versions/layers here.
- `loadDesign()` ā€” `getDoc` on design ref, return null if not exists
- `listDesigns()` ā€” `getDocs` on `users/${userId}/designs` collection, map to DesignMetadata
- `deleteDesign()` ā€” cascade delete: load all versions, for each version delete all layers, then delete version, then delete design. Use `writeBatch` for atomicity (Firestore doesn't cascade delete subcollections). Handle batch limit of 500 writes by chunking.
- `saveVersion()` ā€” `setDoc` on version ref. Strip `layers` from version data (layers go in subcollection). Add `updatedAt: serverTimestamp()` to parent design.
- `loadVersions()` ā€” `getDocs` with `orderBy('versionNumber', 'desc')` and optional `limit`. Return without layers (lazy load).
- `loadVersion()` ā€” `getDoc` on specific version ref. Does NOT auto-load layers.
- `saveLayers()` ā€” batch `setDoc` for each layer in subcollection. Include all layer fields.
- `loadLayers()` ā€” `getDocs` on layers subcollection, ordered by `zIndex asc`
- `savePreferences()` ā€” `setDoc` with merge on preferences doc
- `loadPreferences()` ā€” `getDoc`, return defaults if not exists

Add `index.ts` barrel file in `src/services/storage/` that re-exports all types, the interface, and both adapters.
