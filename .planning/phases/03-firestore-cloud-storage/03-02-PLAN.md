Create `src/services/storage/imageStorageService.ts` for uploading and retrieving generated images.

**Imports:** Use Firebase Storage client SDK (modular v9+):
- `ref, uploadBytes, uploadString, getDownloadURL, deleteObject` from `firebase/storage`
- `getStorage` from `firebase/storage`
- Get storage instance from existing Firebase app in `src/lib/firebase.ts`

**Functions:**

1. `uploadGeneratedImage(userId: string, designId: string, versionId: string, imageData: Blob | ArrayBuffer | Uint8Array): Promise<string>`
   - Path: `generated/${userId}/${designId}/${versionId}/design.png`
   - Metadata: `contentType: 'image/png'`, `cacheControl: 'public, max-age=31536000, immutable'` (1 year CDN cache — images are immutable once generated)
   - Custom metadata: `{ userId, designId, versionId, uploadedAt: new Date().toISOString() }`
   - After upload, call `getDownloadURL()` to get permanent URL with Firebase token
   - Return the download URL

2. `uploadImageFromUrl(userId: string, designId: string, versionId: string, sourceUrl: string): Promise<string>`
   - Fetches image from external URL (e.g., Replicate output URL)
   - Converts response to blob
   - Calls `uploadGeneratedImage()` with the blob
   - Return the Cloud Storage download URL
   - This is the common path: Replicate returns a URL, we fetch and store permanently

3. `getImageUrl(userId: string, designId: string, versionId: string): Promise<string | null>`
   - Construct ref to `generated/${userId}/${designId}/${versionId}/design.png`
   - Call `getDownloadURL()`, return URL
   - Return null if object doesn't exist (catch `storage/object-not-found` error)

4. `deleteImage(userId: string, designId: string, versionId: string): Promise<void>`
   - Construct ref, call `deleteObject()`
   - Silently succeed if object doesn't exist

**Error handling:** All functions wrap in try-catch. Log errors with `[ImageStorage]` prefix. Re-throw with user-friendly message (match existing service error pattern).

**Important:** `uploadImageFromUrl` runs server-side only (fetches external URL). Guard with `typeof window === 'undefined'` check or mark as server-only. For client-side uploads, use `uploadGeneratedImage` directly with blob data.

**Install dependency:** `npm install @google-cloud/tasks`

**Create `src/services/tasks/generationQueue.ts`:**

Queue sharding strategy (from research — avoids GCP 1000 queue limit):
- Hash user ID to 20 buckets: `const bucket = hashCode(userId) % 20`
- Queue name: `generation-bucket-${bucket}`
- Each bucket queue: `maxConcurrentDispatches: 3`, `maxDispatchesPerSecond: 1`

Functions:

1. `hashCode(str: string): number` — simple hash (djb2 or similar) producing positive integer

2. `getQueueName(userId: string): string` — returns `generation-bucket-${hashCode(userId) % 20}`

3. `ensureQueueExists(queueName: string): Promise<void>`
   - Uses `CloudTasksClient.getQueue()` to check existence
   - If NOT_FOUND (error code 5), creates queue with:
     - `rateLimits: { maxConcurrentDispatches: 3, maxDispatchesPerSecond: 1 }`
     - `retryConfig: { maxAttempts: 3 }`
   - If exists, skip (idempotent)

4. `enqueueGenerationTask(userId: string, payload: { prompt: string, parameters: object, designId: string, versionId: string }): Promise<string>`
   - Calls `ensureQueueExists()` first
   - Creates task with:
     - `httpRequest.httpMethod: 'POST'`
     - `httpRequest.url: ${CLOUD_RUN_URL}/api/v1/tasks/generate`
     - `httpRequest.headers: { 'Content-Type': 'application/json' }`
     - `httpRequest.body: Buffer.from(JSON.stringify({ userId, ...payload })).toString('base64')`
     - `httpRequest.oidcToken: { serviceAccountEmail: process.env.TASK_SERVICE_ACCOUNT }`
   - Returns task name (ID)

5. `getQueueStatus(userId: string): Promise<{ pendingTasks: number }>` (optional utility)
   - Lists tasks in user's queue bucket, returns count

**Env vars used:** `GCP_PROJECT_ID`, `GCP_LOCATION` (default 'us-central1'), `CLOUD_RUN_URL`, `TASK_SERVICE_ACCOUNT`

**Create `src/app/api/v1/tasks/generate/route.ts`:**

This is the callback endpoint that Cloud Tasks invokes when processing a generation task.

- Export `POST` handler (Next.js App Router convention)
- Verify request comes from Cloud Tasks: check `X-CloudTasks-TaskName` header exists (Cloud Tasks sets this automatically)
- Also verify OIDC token if in production (defense in depth — Phase 2 middleware handles primary auth)
- Parse body: `{ userId, prompt, parameters, designId, versionId }`
- Call existing generation service (import from replicateService or generationRouter)
- After generation completes, call `uploadImageFromUrl()` from imageStorageService to persist the image
- Update Firestore version document with the Cloud Storage imageUrl (import doc/updateDoc from firebase/firestore)
- Return 200 on success, 500 on failure (Cloud Tasks retries on non-2xx)
- Log with `[TaskHandler]` prefix

**Important:** This route should NOT go through the standard user auth middleware (it's called by Cloud Tasks with OIDC, not user cookies). Add it to middleware.ts exclusion list or handle OIDC verification separately.
