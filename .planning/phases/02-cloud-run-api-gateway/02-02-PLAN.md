<task type="auto">
  <name>Task 1: Firestore quota tracker with per-user enforcement</name>
  <files>src/lib/quota-tracker.ts</files>
  <action>
    Create `src/lib/quota-tracker.ts` with Firestore-based per-user quota tracking:

    1. Define `QuotaConfig` interface: `{ maxRequests: number; windowMs: number }`.

    2. Define `QUOTA_CONFIGS` record with endpoint-specific limits:
       - `'generation'`: 20 requests per hour (60 * 60 * 1000 ms)
       - `'matching'`: 100 requests per hour
       - `'council'`: 20 requests per hour
       - `'upload'`: 50 requests per hour
       - `'default'`: 200 requests per hour (fallback for unlisted endpoints)

    3. Implement `checkQuota(userId: string, endpoint: string): Promise<{ allowed: boolean; remaining: number; retryAfter?: number }>`:
       - Get Firestore instance from `firebase-admin/firestore` (use `getFirestore()`)
       - Quota doc path: `quotas/{userId}/endpoints/{endpoint}`
       - Read current quota document
       - Filter `requests` array to only entries within the current window (`Date.now() - windowMs`)
       - If active requests >= maxRequests, return `{ allowed: false, remaining: 0, retryAfter }` where retryAfter = seconds until oldest request expires
       - Otherwise, append current timestamp to requests array using `FieldValue.arrayUnion(Date.now())`
       - Use `set({ requests: [...filtered, Date.now()], updatedAt: FieldValue.serverTimestamp() }, { merge: true })` to also clean expired entries
       - Return `{ allowed: true, remaining: maxRequests - activeRequests.length - 1 }`

    4. Add a `resetQuota(userId: string, endpoint: string)` function for admin use.

    5. Add graceful fallback: if Firestore is unavailable (e.g., local dev without emulator), log warning and return `{ allowed: true, remaining: -1 }` so development isn't blocked.

    6. Import firebase-admin using: `import { getFirestore, FieldValue } from 'firebase-admin/firestore'`. The Firebase Admin SDK is already initialized from Phase 1 (`src/lib/auth-dal.ts`).
  </action>
  <verify>

    - Contains `getFirestore` import from `firebase-admin/firestore`
    - Contains all 5 quota configs (generation, matching, council, upload, default)
    - Contains graceful fallback for dev environments
  </verify>

  </done>

  <files>src/lib/rate-limit.ts, src/lib/budget-tracker.ts</files>
  <action>
    1. **Rewrite `src/lib/rate-limit.ts`** to delegate to the new quota tracker:
       - Import `checkQuota` from `./quota-tracker`
       - Import `getUserFromRequest` from `./api-auth` (already exists from Phase 1)
       - Map `limitType` parameter to endpoint names: `'generation' | 'matching' | 'council' | 'upload' | 'default'`
       - Extract userId from request using `getUserFromRequest(req)`
       - If no userId (unauthenticated), use IP-based fallback: `req.headers.get('x-forwarded-for') || 'anonymous'`
       - Call `checkQuota(userId, endpoint)` and return the result
       - Keep the same function signature `checkRateLimit(req: NextRequest, limitType: string)` for backward compatibility but change return type to `Promise<{ allowed: boolean; remaining: number; retryAfter?: number }>`
       - Export a helper `rateLimitResponse(result)` that creates a `NextResponse` with status 429, `Retry-After` header, and JSON body `{ error: 'Rate limit exceeded', retryAfter: N }`

    2. **Create `src/lib/budget-tracker.ts`** for server-side Replicate spend tracking:
       - Define `BudgetConfig`: `{ maxSpendCents: number; periodMs: number }`. Default: 50000 cents ($500) per 30 days.
       - Implement `checkBudget(userId?: string): Promise<{ allowed: boolean; spentCents: number; remainingCents: number }>`:
         - Firestore doc path: `budget/global` (global budget for MVP, per-user later)
         - Read current spend document with `spentCents` and `periodStart` fields
         - If `periodStart` is older than `periodMs`, reset the counter
         - If `spentCents >= maxSpendCents`, return `{ allowed: false, spentCents, remainingCents: 0 }`
         - Otherwise return `{ allowed: true, spentCents, remainingCents: maxSpendCents - spentCents }`
       - Implement `recordSpend(amountCents: number): Promise<void>`:
         - Atomically increment `spentCents` field using `FieldValue.increment(amountCents)`
         - Update `lastUpdated` timestamp
       - Add dev fallback: if Firestore unavailable, log warning and allow (same pattern as quota-tracker)
  </action>
