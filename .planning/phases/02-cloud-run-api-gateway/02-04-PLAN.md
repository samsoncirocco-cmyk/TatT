<task type="auto">
  <name>Task 1: Create Cloud Run and API Gateway deployment scripts</name>
  <files>scripts/deploy-cloud-run.sh, scripts/deploy-api-gateway.sh</files>
  <action>
    1. Create `scripts/deploy-cloud-run.sh`:
       - Shebang `#!/usr/bin/env bash`, `set -euo pipefail`
       - Accept optional `--project` and `--region` flags, default to `gcloud config get-value project` and `us-central1`
       - Define service name `tattester-api`, image tag `gcr.io/$PROJECT_ID/tattester:latest`
       - Step 1: Build and push image — `gcloud builds submit --tag $IMAGE_TAG --project $PROJECT_ID`
       - Step 2: Deploy to Cloud Run:
         ```
         gcloud run deploy $SERVICE_NAME \
           --image $IMAGE_TAG \
           --platform managed \
           --region $REGION \
           --min-instances 1 \
           --max-instances 10 \
           --timeout 300 \
           --cpu 2 \
           --memory 1Gi \
           --concurrency 80 \
           --set-env-vars NODE_ENV=production \
           --allow-unauthenticated \
           --project $PROJECT_ID
         ```
       - Note in comments: `--allow-unauthenticated` because API Gateway handles auth. Secrets are added separately via `--update-secrets` after Secret Manager setup.
       - Step 3: Get and print the service URL: `gcloud run services describe $SERVICE_NAME --region $REGION --format 'value(status.url)'`
       - Add progress `echo` statements between steps

    2. Create `scripts/deploy-api-gateway.sh`:
       - Shebang `#!/usr/bin/env bash`, `set -euo pipefail`
       - Accept optional `--project` and `--region` flags
       - Step 1: Get Cloud Run service URL and substitute it into api-spec.yaml:
         ```bash
         CLOUD_RUN_URL=$(gcloud run services describe tattester-api --region $REGION --format 'value(status.url)' --project $PROJECT_ID)
         sed "s|https://tattester-api-HASH-uc.a.run.app|$CLOUD_RUN_URL|g" openapi/api-spec.yaml > /tmp/api-spec-resolved.yaml
         sed -i "s|PROJECT_ID|$PROJECT_ID|g" /tmp/api-spec-resolved.yaml
         ```
       - Step 2: Create or update API config:
         ```bash
         gcloud api-gateway api-configs create tattester-config-$(date +%Y%m%d%H%M) \
           --api=tattester-api \
           --openapi-spec=/tmp/api-spec-resolved.yaml \
           --project=$PROJECT_ID \
           --backend-auth-service-account=tattester-api@$PROJECT_ID.iam.gserviceaccount.com
         ```
       - Step 3: Create or update gateway:
         ```bash
         gcloud api-gateway gateways create tattester-gateway \
           --api=tattester-api \
           --api-config=$CONFIG_NAME \
           --location=$REGION \
           --project=$PROJECT_ID
         ```
       - Handle "already exists" errors gracefully (use `gcloud api-gateway gateways update` as fallback)
       - Step 4: Print gateway URL

    3. Make both scripts executable: add comment at top explaining usage.
  </action>
  <verify>

    - `scripts/deploy-api-gateway.sh` exists and contains `gcloud api-gateway`
    - Both scripts use `set -euo pipefail`
    - Both scripts accept `--project` flag
  </verify>
