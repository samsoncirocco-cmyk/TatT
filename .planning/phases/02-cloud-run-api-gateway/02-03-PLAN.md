  <action>
    Add rate limiting and budget enforcement to all API v1 routes. Each route already has `verifyApiAuth(req)` from Phase 1. Add rate limiting AFTER auth verification.

    **For each route, add immediately after the `verifyApiAuth(req)` call:**

    ```typescript
    import { checkRateLimit, rateLimitResponse } from '@/lib/rate-limit';
    // ... inside handler, after verifyApiAuth:
    const rateResult = await checkRateLimit(req, '{endpoint_type}');
    if (!rateResult.allowed) {
      return rateLimitResponse(rateResult);
    }
    ```

    **Endpoint type mapping:**
    - `generate/route.ts` → `'generation'`
    - `council/enhance/route.ts` → `'council'`
    - `match/semantic/route.ts` → `'matching'`
    - `match/update/route.ts` → `'matching'`
    - `embeddings/generate/route.ts` → `'default'`
    - `ar/visualize/route.ts` → `'default'`
    - `layers/decompose/route.ts` → `'default'`
    - `stencil/export/route.ts` → `'default'`
    - `upload-layer/route.ts` → `'upload'`
    - `storage/upload/route.ts` → `'upload'`
    - `storage/get-signed-url/route.ts` → `'default'`

    **Budget enforcement for generation route ONLY:**
    Add to `src/app/api/v1/generate/route.ts`, after rate limit check and before calling Replicate/Vertex AI:

    ```typescript
    import { checkBudget, recordSpend } from '@/lib/budget-tracker';
    // ... after rate limit check:
    const budgetResult = await checkBudget();
    if (!budgetResult.allowed) {
      return NextResponse.json(
        { error: 'Budget limit reached', spentCents: budgetResult.spentCents },
        { status: 402 }
      );
    }
    ```

    And after successful generation, record the spend:
    ```typescript
    // After successful generation response is ready:
    await recordSpend(estimatedCostCents); // Use 5 cents as default estimate per generation
    ```

    **Do NOT modify any existing business logic.** Only add the rate limit + budget checks. Keep all existing imports, auth checks, and response formatting.
