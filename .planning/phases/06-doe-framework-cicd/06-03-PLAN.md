**src/app/api/health/startup/route.ts:**
Create a Next.js API route that validates environment readiness. This is the TypeScript equivalent of validate_env.py, designed for Cloud Run's startup probe.

The endpoint must:
- Export async GET handler
- Check required env vars exist: GCP_PROJECT_ID, NEO4J_URI, NEO4J_USER, NEO4J_PASSWORD, GCS_BUCKET_NAME
- Attempt Firestore connectivity (import firebase-admin, call `getFirestore().collection('_health_check').doc('startup').set(...)`)
- Attempt Neo4j connectivity (import neo4j driver, run `RETURN 1 AS num`)
- Return JSON: `{ status: "healthy"|"unhealthy", checks: [{service: string, healthy: boolean, message: string}], timestamp: ISO string }`
- Return 200 if all checks pass, 503 if any check fails
- Wrap each check in try/catch so one failure doesn't prevent other checks from running
- Include a timeout of 10 seconds per check (use Promise.race with a timeout promise)
- Skip checks gracefully if env vars for that service aren't set (mark as "skipped" not "failed" — supports local dev)

**Dockerfile:**
Enhance the existing Dockerfile (already has 3 stages: deps, builder, runner) to add a Python stage. Follow the Pattern 5 from 06-RESEARCH.md.

Changes to existing Dockerfile:
1. Keep existing deps, builder stages unchanged
2. Add new stage AFTER builder, BEFORE runner:
   ```
   FROM python:3.12-slim AS python-builder
   WORKDIR /app
   COPY execution/requirements.txt ./execution/
   RUN python -m venv /venv && /venv/bin/pip install --no-cache-dir -r execution/requirements.txt
   ```
3. In the runner stage, add Python runtime:
   ```
   RUN apk add --no-cache python3 py3-pip
   ```
4. Copy Python venv and execution scripts:
   ```
   COPY --from=python-builder --chown=nextjs:nodejs /venv /venv
   COPY --chown=nextjs:nodejs execution/ ./execution/
   COPY --chown=nextjs:nodejs directives/ ./directives/
   ```
5. Add PATH for Python venv: `ENV PATH="/venv/bin:$PATH"`
6. Add HEALTHCHECK instruction (as shown in research)

Keep all existing NEXT_PUBLIC_* ARGs and ENV vars unchanged. Keep existing non-root user setup unchanged.

Create `.github/workflows/` directory and `ci-cd.yml`. Follow Pattern 3 from 06-RESEARCH.md with these specifics:

**Triggers:**
- `push` to `main` branch (full pipeline: lint + test + build + deploy)
- `pull_request` to `main` branch (lint + test only, no build/deploy)
- `workflow_dispatch` (manual trigger for deployments)
- Path filters: only trigger on changes to `src/**`, `execution/**`, `tests/**`, `package*.json`, `Dockerfile`, `.github/workflows/**`, `directives/**`

**Jobs:**

1. **lint** (runs always):
   - actions/checkout@v4
   - actions/setup-node@v4 with node-version 20 and cache 'npm'
   - `npm ci --legacy-peer-deps`
   - `npm run lint`

2. **test-js** (runs always, parallel with lint):
   - actions/checkout@v4
   - actions/setup-node@v4 with node-version 20 and cache 'npm'
   - `npm ci --legacy-peer-deps`
   - `npm run test`
   - Upload coverage artifact (actions/upload-artifact@v4)

3. **test-python** (runs always, parallel with lint and test-js):
   - actions/checkout@v4
   - actions/setup-python@v5 with python-version 3.12 and cache 'pip'
   - `pip install -r execution/requirements.txt`
   - `pytest tests/execution/ -v --cov=execution --cov-report=xml`
   - Upload python coverage artifact

4. **build** (needs: [lint, test-js, test-python], runs on push to main or workflow_dispatch):
   - actions/checkout@v4
   - docker/setup-buildx-action@v3
   - actions/cache@v4 for Docker layers at /tmp/.buildx-cache
   - google-github-actions/auth@v2 with Workload Identity Federation
     - workload_identity_provider: `${{ secrets.WIF_PROVIDER }}`
     - service_account: `${{ secrets.WIF_SERVICE_ACCOUNT }}`
   - Configure Docker for GCR: `gcloud auth configure-docker`
   - docker/build-push-action@v5:
     - push: true (to GCR)
     - tags: `gcr.io/${{ secrets.GCP_PROJECT_ID }}/pangyo:${{ github.sha }}`, `gcr.io/${{ secrets.GCP_PROJECT_ID }}/pangyo:latest`
     - cache-from/cache-to with local cache
     - build-args for NEXT_PUBLIC_* vars from secrets
   - Cache rotation (rm old, mv new)

5. **deploy** (needs: build, runs only on push to main, NOT on PRs):
   - Condition: `if: github.ref == 'refs/heads/main' && github.event_name != 'pull_request'`
   - actions/checkout@v4
   - google-github-actions/auth@v2 (same WIF config)
   - google-github-actions/get-secretmanager-secrets@v2 for runtime secrets:
     - REPLICATE_API_TOKEN from Secret Manager
     - NEO4J_PASSWORD from Secret Manager
     - OPENROUTER_API_KEY from Secret Manager
   - google-github-actions/deploy-cloudrun@v2:
     - service: pangyo-production
     - region: us-central1
     - image: `gcr.io/${{ secrets.GCP_PROJECT_ID }}/pangyo:${{ github.sha }}`
     - flags: `--cpu=2 --memory=2Gi --max-instances=10 --min-instances=1`
     - env_vars: NODE_ENV=production, GCP_PROJECT_ID=${{ secrets.GCP_PROJECT_ID }}
     - Secrets passed via secrets parameter (runtime injection, NOT baked into image)

**Required GitHub Secrets** (document as YAML comments at top of file):
