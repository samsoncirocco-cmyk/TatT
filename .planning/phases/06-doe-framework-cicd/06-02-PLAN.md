**execution/seed_artists.py:**
Import artist data into Neo4j from a JSON file.
- Accept `--input` (path to JSON file), `--batch-size` (default 100), `--dry-run` flag
- JSON format: array of objects with `{name, styles: [], location, portfolio_url, specialties: []}`
- Create Cypher queries: MERGE Artist nodes, CREATE Style nodes, CREATE relationships (HAS_STYLE, SPECIALIZES_IN)
- Use Neo4j driver with connection from env vars (NEO4J_URI, NEO4J_USER, NEO4J_PASSWORD)
- Print progress: "Imported {n}/{total} artists"
- On --dry-run: parse and validate JSON, print what would be imported, don't connect to Neo4j

**execution/generate_embeddings.py:**
Generate Vertex AI text-embedding-004 embeddings for artist portfolios.
- Accept `--project-id` (GCP project), `--batch-size` (default 10), `--delay` (seconds between batches, default 1.0), `--dry-run` flag
- Read artist data from Firestore collection `artists` (or accept `--source neo4j` to read from Neo4j)
- For each artist: concatenate name + styles + specialties into text, generate embedding via Vertex AI
- Store embeddings back in Firestore document: `artists/{artist_id}/embedding` field (768-dim float array)
- Respect rate limits: sleep `--delay` seconds between batches
- Print progress: "Generated {n}/{total} embeddings"
- On --dry-run: show what text would be embedded, don't call Vertex AI

**execution/check_budget.py:**
Check Replicate API spend against $500 budget.
- Accept `--budget` (default 500.0), `--warn-threshold` (default 0.75 = 75%)
- Read generation events from Firestore collection `generation_events` (or fallback: count documents in `users/*/designs/*/versions/*` as proxy)
- Calculate estimated spend based on per-model costs: SDXL=$0.02/gen, Imagen=$0.05/gen
- Print summary: "Estimated spend: ${amount} / ${budget} ({percent}%)"
- Exit 1 if spend exceeds --warn-threshold percentage
- Support `--json` flag for machine-readable output

**execution/migrate_localStorage.py:**
Migrate exported browser localStorage data to Firestore.
- Accept `--input` (path to JSON export file), `--user-id` (Firestore user ID), `--dry-run` flag
- JSON format: the version_history_${sessionId} structure from versionService.js
- Map to Firestore subcollections: `users/{uid}/designs/{designId}/versions/{versionId}/layers/{layerId}`
- Handle image URLs: if URL is a data: URI, upload to Cloud Storage and replace with permanent URL
- Print progress: "Migrated {n}/{total} designs"
- On --dry-run: parse JSON, print mapping summary, don't write to Firestore
