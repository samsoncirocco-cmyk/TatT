Each directive MUST have these sections:
- **Purpose** - What and why
- **Prerequisites** - Checklist of conditions
- **Procedure** - Numbered steps with bash commands referencing execution/ scripts
- **Rollback** - How to undo
- **Known Issues** - Empty section with template for self-annealing (placeholder comment: "No known issues yet. Update this section when issues are discovered.")
- **Post-Operation** - Checklist including "update Known Issues if any problems occurred"
- **Related Directives** - Cross-references

Directive-specific content:

1. **deploy.md**: Cloud Run deployment via GitHub Actions (preferred) or manual `gcloud run deploy`. References `run_health_checks.py` for post-deploy verification. Includes staging vs production distinction. Mentions secrets injection from Secret Manager (not env files).

2. **seed-artists.md**: Run `python execution/seed_artists.py` to import artist data into Neo4j from JSON source. Prerequisites: Neo4j connection, source data file. Includes batch size guidance (100 records/batch) and rate limit awareness.

3. **generate-embeddings.md**: Run `python execution/generate_embeddings.py` for Vertex AI text-embedding-004 embeddings. Prerequisites: GCP project ID, Vertex AI API enabled. Batch processing with --batch-size and --delay flags. Cost awareness: ~$0.025 per 1000 embeddings.

4. **migrate-data.md**: Run `python execution/migrate_localStorage.py` to migrate browser localStorage exports to Firestore. Prerequisites: Firestore access, exported JSON file. Covers the `users/{uid}/designs/{did}/versions/{vid}/layers/{lid}` subcollection structure.

5. **monitor-budget.md**: Run `python execution/check_budget.py` for Replicate API spend tracking. Covers GCP billing API queries, alert thresholds at 50%/75%/90% of $500 budget, and where to view Cloud Monitoring dashboards.

6. **onboard-engineer.md**: Complete new engineer onboarding path. Steps: (1) Clone repo, (2) Install Node.js 20 + Python 3.12, (3) Copy .env.example and fill values, (4) Run `python execution/validate_env.py` to verify all services reachable, (5) `npm install && npm run dev`, (6) Run tests `npm test`, (7) Deploy to staging. Target: productive in under 2 hours. References CLAUDE.md for architecture overview.

7. **rotate-secrets.md**: Secret Manager key rotation procedure. Lists all secrets (replicate-api-token, neo4j-password, firebase-private-key, openrouter-api-key). Steps: create new version in Secret Manager, redeploy Cloud Run to pick up new version, verify with health checks, disable old version after 24h grace period.

Use consistent formatting: H1 for title, H2 for sections, H3 for subsections and Known Issue entries. Include `Last Updated: 2026-02-16` and `Last Tested: Not yet tested` in metadata at top of each directive.

**execution/validate_env.py:**
Follow the detailed pattern from 06-RESEARCH.md (Python Environment Validation Script section). The script must:
- Check required env vars: GCP_PROJECT_ID, NEO4J_URI, NEO4J_USER, NEO4J_PASSWORD, GCS_BUCKET_NAME
- Check Secret Manager accessibility for: replicate-api-token, neo4j-password, firebase-private-key, openrouter-api-key
- Check Firestore connectivity (write/read health check doc to `_health_check` collection)
- Check Neo4j connectivity (run `RETURN 1 AS num` query)
- Check Cloud Storage bucket existence
- Print clear pass/fail for each check with service name
- Exit 0 if all pass, exit 1 if any fail
- Support `--skip` flag to skip specific checks (e.g., `--skip neo4j` for local dev without Neo4j)
- Support `--json` flag to output results as JSON (for startup probe consumption)
- Use type hints throughout, Python 3.12 compatible
- Each check function returns Tuple[bool, str] (healthy, message)
- Wrap all external calls in try/except to prevent unhandled crashes

**execution/run_health_checks.py:**
Smoke test script that hits running service endpoints:
- Accept `--base-url` argument (default: http://localhost:3000)
- Check GET /api/health returns 200
- Check GET /api/health/startup returns 200 (once startup endpoint exists)
- Check POST /api/neo4j/query with a simple RETURN 1 query
- Print pass/fail summary
- Exit 0 if all pass, exit 1 if any fail
- Support `--check` flag to run specific check only
- Use `requests` library for HTTP calls
- 10 second timeout per request
